### 4.3 🔴 性能建议

| # | 优先级 | 建议 | 预期收益 |
|---|--------|------|---------|
| 1 | **高** | **Element Plus 按需导入** — 当前是全量导入，使用 `unplugin-element-plus` 或手动按需导入可将 JS 从 1.1MB 降至约 250-400KB | JS 减少 60-70% |
| 2 | 中 | **CSS 代码分割** — Element Plus CSS 也应按需加载，352KB 中大部分组件未使用 | CSS 减少 60-80% |
| 3 | 中 | **ECharts 按需导入** — 如果只有少数图表场景，按需导入 ECharts 模块 | JS 减少 ~800KB (含 ECharts) |
| 4 | 低 | **Gzip/Brotli 压缩** — 生产部署启用压缩可进一步减少传输大小 | 传输量减少 60-70% |

### 建议测试金字塔

```
       ┌──────┐
       │ E2E  │  3-5 个核心旅程
       │ 10%  │  (登录→聊天→查看历史)
       ├──────┤
       │集成  │  8-12 个组件交互测试
       │ 20%  │  (ChatBox ↔ InputPanel, Mock 服务)
       ├──────┤
       │单元  │  30+ 个单元测试
       │ 70%  │  (utils, api mocks, 组件纯逻辑)
       └──────┘
```

### 推荐优先编写的测试

1. **单元测试（最高优先级）**
   - `chatMock.js` — 模拟响应生成逻辑（不同输入→不同步骤+工具调用）
   - `request.js` — 请求/响应拦截器
   - `config/index.js` — 环境配置读取

2. **E2E 测试（使用 Playwright）**
   ```gherkin
   Scenario: 首页导航到聊天
     Given 用户在首页
     When 点击 "开始对话"
     Then 页面跳转到 /chat
     And 显示聊天输入框和示例问题

   Scenario: 发送消息获得响应
     Given 用户在聊天页面
     When 输入 "北京今天天气怎么样？"
     And 按 Enter 发送
     Then 显示用户消息气泡
     And 显示 Agent 回答（含步骤时间线）
     And 显示工具调用记录

   Scenario: 历史记录页面
     Given 用户在历史记录页面
     When 无已有会话
     Then 显示空状态提示
     And "清空全部" 按钮处于禁用状态
   ```
